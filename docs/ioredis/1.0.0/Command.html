new Command(name, args, callback)

command.js, line 31

Command instance

It's rare that you need to create a Command instance yourself.

Name Type Default Description
name string

Command name
args Array.<string> null optional

An array of command arguments
callback function null optional

The callback that handles the response. If omit, the response will be handled via Promise.
See:
  • Redis#sendCommand which can send a Command instance to Redis
Example
var infoCommand = new Command('info', null, function (err, result) {
  console.log(result);
});

redis.sendCommand(infoCommand);

var getCommand = new Command('get', ['foo']);

// getCommand don't have a callback, so Redis#sendCommand will return a Promise.
redis.sendCommand(getCommand).then(function (result) {
  console.log(result);
});

resolve(value)

command.js, line 76

Return a value to the callback

This method and Command#reject invoke the callback in the next event loop via setImmediate to keep the order of the results when mix using the callback-style and promise-style. For instance:

redis.get('foo').then(function () {
  console.log('this comes first');
});
redis.get('bar', function () {
  console.log('this comes second');
});

Without setImmediate, the order will reverse.
